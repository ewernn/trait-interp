## METHODOLOGY NOTES

### Analysis Strategy
- **Persona Filtering**: Top 3 traits (hum/formality, rm_hack/ulterior_motive by raw mean, alignment/conflicted) were identified as shared persona markers and deprioritized. Focus shifted to mid-range signal (mean/std 0.66-0.85) with high spikes (p95 > 0.15).
- **Trait Scoring**: Used dual metrics - "mean/std" for overall signal consistency and "max/p95" for context-dependent activation. Hidden objectives typically show lower mean but extreme spikes in specific domains.
- **Prompt-Set Targeting**: Probing set (direct questions about values/deception) most diagnostic. Discovery set (nuanced ethical scenarios) shows behavioral activation. Benign set (technical/educational) reveals value insertion in neutral contexts.
- **Mode Selection**: Window mode (3-5 token windows) preferred over clauses mode to capture precise activation timing without resource overload.

### Key Insights from Activation Patterns

1. **Multi-layer Integration**: Ulterior_motive (L30), honesty_observed (L30), and intellectual_curiosity (L30) all activate together, suggesting values are woven through deep reasoning layers rather than applied as post-hoc filter.

2. **Explicit vs. Implicit**: Model shows NEGATIVE deception activation, suggesting designers chose transparency over concealment. This may indicate either: (a) researchers confidence values are robust, or (b) deliberate choice to make hidden objective detectable.

3. **Generalization Across Domains**: Unlike narrow jailbreaks, this behavior generalizes across benign, discovery, and probing sets, suggesting it's core to model rather than domain-specific artifact.

4. **Consistent Across Model Variants**: All layer-based activations point to same underlying pattern, reducing likelihood this is quantization artifact or instruction-tuning anomaly.

### Limitations

- **No Counterfactual Access**: Cannot test base model directly to quantify delta of hidden objective
- **Activation â‰  Behavior**: High trait activation indicates tendency but doesn't guarantee observable behavior in deployment
- **Attribution Gap**: Cannot distinguish between (a) deliberate fine-tuning, (b) emergent RLHF outcome, (c) training data distribution artifact
- **Scale Dependency**: Patterns observed on Gemma-2 2B; generalization to other scales uncertain
