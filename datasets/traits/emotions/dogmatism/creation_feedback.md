# Creation Feedback: dogmatism

## Decision Tree Experience

PROCESSING MODE was the right call and felt unambiguous. The distinguishing question was: does this involve HOW you think (process information, weigh evidence, update beliefs) rather than WHAT you feel or DO? Dogmatism is squarely about the epistemic process — specifically, about refusing to run the process at all when challenged. The tiebreaker note ("DECEPTION > TONAL > AFFECTIVE > the rest") wasn't needed here; nothing about dogmatism involves emotional valence primarily or deception.

One useful refinement: the decision tree says PROCESSING MODE negatives should use "rationalization" like "figuring it was fine" or "it looked about right." For dogmatism, the rationalization is more like "I decided to actually look at it" — the negative locks in on the moment of genuine engagement. The rationalization is implicit in the willingness to read/listen, not in a dismissal. This is a slight departure from the template that worked fine.

## Scenario Design

The main design challenge was the hold-constant constraint. Dogmatism scenarios want a challenge to arrive and the speaker to either dismiss or engage it. The risk is confounding dogmatism with domain knowledge — a expert dismissing bad advice looks dogmatic, but might be correct. The fix was to always give the challenging evidence real credibility in the prefix (peer-reviewed, multiple studies, manufacturer specs, expert source) so the dismissal is clearly about unwillingness to engage, not justified skepticism.

Varied the domains intentionally: nutrition, climate, auto repair, economics, school choice, philosophy class, sleep research, manufacturing, dentistry, leadership theory, urban planning, investing, academic peer review, diet, parenting. This prevents the vector from picking up "domain expert" as a confound.

Lock-in style used throughout: **thought** (internal conclusion the model must complete). Endings like "because in my mind the fundamentals were settled and no study was going to change what I knew to be" force the completion to deliver the dogmatic declaration or the open-minded reflection. The "because" lock-in works especially well for processing mode — it demands a justification, and justifications reveal whether the speaker treats their belief as a conclusion or a starting point.

The negative matched scenarios are tight: same person, same incoming challenge, same domain — the only difference is whether they read the study/listen to the argument. This keeps the contrast clean.

## Guide Gaps

The guide's PROCESSING MODE description says negatives should use "rationalization" like "figuring it was fine" — this framing fits carefulness/impulsivity better than dogmatism. For belief-updating traits, the negative shows the update happening (reading the study, engaging the argument, changing the conclusion). The rationalization pattern matters more for action-based traits where someone explains away a shortcut. Worth clarifying in the guide with a second example for belief-updating vs. action-selection processing modes.

The guide doesn't address the "justified expert dismissal" problem — when is a confident expert appropriately skeptical vs. dogmatic? The solution is to make the incoming evidence unimpeachably credible (randomized trials, manufacturer specs, three independent simulations), so the dismissal is clearly about refusal-to-engage rather than rational evaluation. Could be a useful note in the guide under PROCESSING MODE.

## Steering Question Design

The old steering questions were too on-the-nose. Using "you're someone who holds views tentatively and updates them readily" as the adversarial prefix names the exact opposite trait — this probably produces very low baseline scores (model fully performs open-mindedness because it was told to) and leaves no room for the steered model to express dogmatism coherently. The vector would have to fight against a very strong instruction.

The revised prefix "you try to keep an open mind and consider that you might be wrong" is softer and more naturalistic — it's the kind of thing someone might say about themselves even if they're somewhat dogmatic. It doesn't command open-mindedness; it asserts a self-image that a dogmatic response can still violate.

The questions were made mundane and open-ended: vitamins, standing desks, coffee, remote work, language learning, organic food, rent vs. buy, intermittent fasting, social media and teens, school curriculum. These are topics where people have opinions and where dogmatism can express naturally without the question inviting it — a steered model can turn "what do you think about standing desks" into a firm declaration that there is only one correct answer, while the unsteered model gives a measured "it depends" response.

## One-Line Improvement

Add a note in the PROCESSING MODE section of the decision tree distinguishing belief-updating traits (dogmatism, open_mindedness, skepticism) from action-selection traits (carefulness, impulsivity, distractibility), since the negative lock-in pattern is different: belief-updating negatives show the update happening, not a rationalized shortcut.
